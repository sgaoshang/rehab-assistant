# 语音播报功能说明

## 当前实现状态

### ✅ 已实现的功能

1. **应用打开时的自动播报**
   - 当应用在前台运行时，收到通知会自动语音播报
   - 播报内容包括项目名称和描述

2. **点击通知后播报**
   - 用户点击通知打开应用后会自动播报
   - 延迟 1 秒确保应用完全打开

3. **锁屏通知优化**（v1.0.1 新增）
   - 通知优先级设为最高（MAX）
   - 支持锁屏显示（PUBLIC visibility）
   - 增强振动和灯光提示
   - 添加交互式操作按钮

4. **通知操作按钮**（v1.0.1 新增）
   - 🔊 播报：在锁屏状态下可点击此按钮打开应用并播报
   - ✓ 完成：快速关闭通知

### ⚠️ 技术限制

#### 为什么锁屏时不能自动语音播报？

**Android 系统限制**：
- 锁屏时应用进入后台睡眠状态
- JavaScript 代码停止执行
- `expo-speech` 需要应用在前台才能工作
- Android 8.0+ 对后台服务有严格限制

**Expo Managed Workflow 限制**：
- 无法直接添加原生代码
- 无法访问 Android 原生 TextToSpeech API
- 需要前台服务才能在后台播放音频

## 使用建议

### 当前最佳使用方式

1. **应用打开状态**：通知会自动播报 ✅
2. **锁屏状态**：
   - 方式 1：点击通知打开应用，自动播报 ✅
   - 方式 2：在通知上点击"🔊 播报"按钮 ✅
   - 方式 3：解锁手机后再打开应用 ✅

### 配置建议

为了确保在锁屏状态下收到通知：

1. **关闭勿扰模式**（如果需要在勿扰时段接收）
2. **设置中允许锁屏通知显示**
3. **允许应用在后台运行**（部分手机需要）
4. **关闭电池优化**（部分手机可能需要）

## 未来改进方案

### 方案 1：切换到 Bare Workflow
- 优点：可以添加原生 Android TTS 模块
- 缺点：失去 Expo Managed 的便利性
- 工作量：中等

### 方案 2：使用 Expo Config Plugin
- 优点：保持 Managed Workflow
- 缺点：需要开发自定义插件
- 工作量：较大

### 方案 3：使用预录音频 + expo-av
- 优点：技术上可行
- 缺点：需要录制大量音频文件，不够灵活
- 工作量：较大

### 方案 4：使用前台服务
- 优点：可以在后台播报
- 缺点：会显示持久通知，用户体验不佳
- 工作量：中等

## 技术细节

### 当前权限配置

```json
{
  "android": {
    "permissions": [
      "RECEIVE_BOOT_COMPLETED",  // 开机自启
      "VIBRATE",                 // 振动
      "SCHEDULE_EXACT_ALARM",    // 精确定时
      "USE_FULL_SCREEN_INTENT",  // 全屏通知
      "WAKE_LOCK",               // 唤醒屏幕
      "FOREGROUND_SERVICE"       // 前台服务（预留）
    ]
  }
}
```

### 通知配置

```javascript
{
  importance: MAX,                    // 最高优先级
  lockscreenVisibility: PUBLIC,       // 锁屏可见
  bypassDnd: true,                    // 绕过勿扰
  categoryIdentifier: 'reminder',     // 支持操作按钮
}
```

## 常见问题

**Q: 为什么不像闹钟应用那样在锁屏时播报？**
A: 闹钟应用使用的是 `AlarmManager` + 全屏 Activity，这是原生 Android 功能。Expo Managed Workflow 无法直接使用这些原生 API。

**Q: 能否使用通知声音来播报？**
A: 可以，但需要预录音频文件，无法动态生成语音内容。

**Q: 什么时候会支持锁屏自动播报？**
A: 需要切换到 Bare Workflow 或等待 Expo 提供相关插件支持。
