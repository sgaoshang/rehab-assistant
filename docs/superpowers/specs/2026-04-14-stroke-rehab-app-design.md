# 脑梗康复训练辅助APP设计文档

**日期：** 2026-04-14  
**项目名称：** 康复助手  
**目标平台：** iOS & Android

## 项目概述

### 目标用户
- **主要用户：** 脑梗康复期患者（老年人）
- **次要用户：** 家属/护理人员（协助设置）

### 核心需求
为脑梗患者提供简单易用的康复训练提醒和打卡应用，帮助患者坚持规律的康复训练，记录康复进展。

### 成功标准
1. 老年患者能独立完成日常打卡操作
2. 提醒功能准确可靠
3. 界面清晰，字体够大，操作简单
4. 数据可导出分享给家属

## 技术方案

### 技术栈选择

**方案：Expo + 部分原生功能（平衡方案）**

**核心技术：**
- **框架：** React Native + Expo SDK 51+
- **语言：** TypeScript
- **导航：** React Navigation 6.x
- **状态管理：** React Context API
- **本地存储：** AsyncStorage
- **通知系统：** Expo Notifications
- **日期处理：** date-fns
- **日历组件：** react-native-calendars

**选择理由：**
1. **开发效率高：** Expo 提供完整的开发工具链，无需配置复杂的原生环境
2. **跨平台一致：** 一套代码同时支持 iOS 和 Android
3. **功能完整：** Expo Notifications 满足定时提醒需求，AsyncStorage 满足本地存储需求
4. **易于部署：** 可直接构建 APK/IPA，无需上架应用商店
5. **扩展性好：** 未来如需云端同步，可平滑升级

### 数据架构

**单向数据流：**
```
用户操作 → Context Actions → Storage层 → AsyncStorage
                                ↓
                          更新Context State
                                ↓
                          重新渲染UI
```

**本地存储键值：**
- `@rehab_app:exercises` - 训练列表（JSON数组）
- `@rehab_app:checkins` - 打卡记录（JSON数组）
- `@rehab_app:settings` - 应用设置（JSON对象）
- `@rehab_app:initialized` - 初始化标记（布尔值）

## 功能设计

### 1. 训练管理

#### 1.1 预设训练模板

**上肢训练：**
1. **握拳练习**
   - 说明：慢慢握紧拳头，保持5秒，然后放松
   - 建议：每次重复10-15次

2. **手指伸展**
   - 说明：将手指尽量张开，保持5秒后放松
   - 建议：每次重复10次

3. **抬臂运动**
   - 说明：慢慢将手臂向前抬高，尽量到肩膀高度
   - 建议：每侧重复8-10次

4. **肩关节旋转**
   - 说明：缓慢转动肩膀，画圆圈
   - 建议：顺时针和逆时针各10次

**下肢训练：**
5. **踝关节运动**
   - 说明：坐姿，脚尖向上勾，再向下压
   - 建议：每侧重复15次

6. **膝关节屈伸**
   - 说明：坐姿，慢慢伸直腿，再弯曲
   - 建议：每侧重复10次

7. **原地踏步**
   - 说明：扶稳椅背，原地缓慢踏步
   - 建议：持续3-5分钟

**综合训练：**
8. **颈部转动**
   - 说明：慢慢左右转头，不要用力过猛
   - 建议：每侧重复8次

9. **深呼吸练习**
   - 说明：深吸气5秒，慢慢呼出5秒
   - 建议：重复10次

10. **平衡训练**
    - 说明：扶稳物体，单脚站立保持10秒
    - 建议：每侧重复5次

#### 1.2 自定义训练

用户可添加自定义训练项目，字段包括：
- 训练名称（必填，最多20字）
- 训练说明（选填，最多100字）
- 提醒时间（可设置多个）

#### 1.3 训练属性

```typescript
interface Exercise {
  id: string;              // UUID
  name: string;            // 训练名称，最多20字
  description: string;     // 说明，最多100字
  isPreset: boolean;       // 是否预设模板
  isEnabled: boolean;      // 是否启用
  reminderTimes: string[]; // 提醒时间数组，格式 "HH:mm"
  createdAt: number;       // 创建时间戳
}
```

### 2. 提醒系统

#### 2.1 通知类型

**主通知（到点提醒）：**
- 标题：「该做训练了」
- 内容：「{训练名称}」
- 副标题：「{训练说明前30字}」
- 声音：温和的系统提示音
- 点击行为：打开APP并导航到该训练的打卡页面

**温馨提醒（超时未完成）：**
- 如果30分钟内未打卡，发送温馨提醒
- 标题：「别忘了哦」
- 内容：「{训练名称} 还没完成呢」

#### 2.2 通知调度逻辑

1. **初始调度：**
   - 应用启动时，遍历所有启用的训练
   - 为每个提醒时间创建通知

2. **通知标识符：**
   - 格式：`{exerciseId}_{HHmm}`
   - 便于更新和取消特定通知

3. **更新策略：**
   - 修改训练时，先取消旧通知，再创建新通知
   - 禁用训练时，取消所有相关通知
   - 删除训练时，取消所有相关通知

4. **权限处理：**
   - 首次启动时请求通知权限
   - 引导文案：「需要您允许通知，这样才能按时提醒您做训练」
   - 如果拒绝，在设置页提供引导

#### 2.3 跨日期调度

- 每天凌晨0点，重新调度当天的所有通知
- 确保通知始终准确

### 3. 打卡功能

#### 3.1 打卡界面

**布局：**
```
┌─────────────────────────────┐
│      [训练名称 28pt]         │
│                             │
│   [训练说明 20pt，灰色]     │
│                             │
│   ┌─────────────────────┐   │
│   │  选择您的感受：      │   │
│   └─────────────────────┘   │
│                             │
│   ┌─────┐ ┌─────┐          │
│   │ 😊  │ │ 🙂  │          │
│   │轻松 │ │正常 │          │
│   └─────┘ └─────┘          │
│                             │
│   ┌─────┐ ┌─────┐          │
│   │ 😕  │ │ 😣  │          │
│   │困难 │ │疼痛 │          │
│   └─────┘ └─────┘          │
│                             │

```

**交互流程：**
1. 点击感受按钮，按钮高亮
2. （可选）输入备注
3. 点击「完成打卡」
4. 保存打卡记录
5. 显示鼓励语（随机）：
   - "太棒了！坚持就是胜利！"
   - "您真棒！继续加油！"
   - "做得好！明天继续哦！"
6. 自动返回主页

#### 3.2 打卡记录数据模型

```typescript
interface CheckIn {
  id: string;              // UUID
  exerciseId: string;      // 关联的训练ID
  exerciseName: string;    // 训练名称（冗余存储）
  timestamp: number;       // 打卡时间戳
  feeling: 'easy' | 'normal' | 'hard' | 'pain'; // 感受
  note?: string;           // 备注（选填）
}
```

#### 3.3 打卡状态判断

**判断逻辑：**
- 获取当天的起始和结束时间戳
- 查询 `checkins` 数组中是否存在：
  - `exerciseId` 匹配
  - `timestamp` 在当天范围内
- 如果存在，标记为已完成

### 4. 历史记录

#### 4.1 统计卡片

**连续打卡天数：**
- 从今天往前查，连续有打卡记录的天数
- 如果中断，重新计数
- 显示：「已坚持 {N} 天」

**本周完成率：**
- 计算本周（周一到周日）应完成的训练次数
- 计算实际完成次数
- 显示：「本周完成率 {X}%」

#### 4.2 日历视图

- 使用 `react-native-calendars` 组件
- 有打卡记录的日期标记绿点
- 点击日期，下方列表跳转到对应日期

#### 4.3 打卡列表

- 显示最近30天的打卡记录
- 按日期倒序排列
- 每条记录显示：
  - 日期
  - 训练名称
  - 感受图标
  - 备注（如有）

## 实现细节

### 通知权限流程

**首次启动：**
1. 检查通知权限状态
2. 如果未授权，显示引导对话框：
   ```
   ┌─────────────────────────────┐
   │                             │
   │  需要您允许通知              │
   │                             │
   │  这样才能按时提醒您做训练    │
   │                             │
   │  ┌──────┐    ┌──────┐      │
   │  │ 稍后 │    │ 允许 │      │
   │  └──────┘    └──────┘      │
   └─────────────────────────────┘
   ```
3. 点击「允许」，调用系统权限请求

**如果用户拒绝：**
- 在设置页显示「开启通知」按钮
- 点击后引导用户到系统设置

### 数据初始化

**首次启动流程：**
1. 检查 `@rehab_app:initialized` 标记
2. 如果未初始化：
   - 创建10个预设训练（默认禁用）
   - 保存到 `@rehab_app:exercises`
   - 设置初始化标记
3. 加载训练列表和打卡记录到 Context

### 错误处理

**AsyncStorage 操作失败：**
- 捕获异常，显示友好提示：「数据保存失败，请重试」
- 记录错误日志（开发环境）

**通知调度失败：**
- 捕获异常，显示提示：「提醒设置失败，请检查通知权限」
- 提供跳转到设置页的按钮

**数据导出失败：**
- 显示提示：「导出失败，请重试」

**所有错误提示：**
- 使用简单语言，避免技术术语
- 提供明确的解决方案或操作建议

### 性能优化

**列表渲染：**
- 使用 `FlatList` 替代 `ScrollView`
- 启用 `windowSize` 优化

**数据缓存：**
- Context 中缓存训练列表和打卡记录
- 修改时同步更新 Context 和 AsyncStorage

**通知调度：**
- 仅在训练修改时重新调度通知
- 避免不必要的通知取消和创建

## 技术风险与应对

### 风险1：通知在后台不可靠
**影响：** iOS/Android 可能杀掉后台进程，导致通知不发送

**应对：**
- 使用 Expo 的本地通知（Local Notifications），不依赖后台任务
- 每天凌晨重新调度当天通知，确保准确性
- 在设置页添加「测试通知」功能，让用户验证

### 风险2：老年用户误操作
**影响：** 可能误删训练、误关通知

**应对：**
- 所有危险操作（删除、清空数据）需要二次确认
- 提供「恢复预设训练」功能
- 删除训练时保留历史打卡记录

### 风险3：应用被系统杀掉后数据丢失
**影响：** 数据可能未及时保存

**应对：**
- 所有修改立即写入 AsyncStorage，不等待
- 关键操作（打卡、添加训练）使用 `await` 确保写入成功
- 添加数据校验，防止损坏

### 风险4：iOS安装复杂
**影响：** 非开发者账号安装IPA较困难

**应对：**
- 优先使用 TestFlight（需要 Apple Developer 账号，99美元/年）
- 或使用 AltStore、Sideloadly 等工具侧载
- 提供详细的安装指南
